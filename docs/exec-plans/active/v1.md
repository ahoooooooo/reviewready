# v1 execution plan

## Goal

Deliver a publishable Node 24 CLI and JavaScript GitHub Action that implement the
v1 product contract with deterministic behavior and a documented security boundary.

## Delivery loop

For each vertical slice:

1. record acceptance examples;
2. add a failing focused test;
3. implement the smallest behavior;
4. run focused and full tests;
5. inject malformed and adversarial inputs;
6. fix defects and retain them as regression tests;
7. update public documentation and this decision log.

## Milestones

- [x] Product contract and architecture boundary.
- [x] Minimal project-local Codex baseline.
- [x] Toolchain and reproducible quality gate.
- [x] Closed policy schema and semantic validation.
- [x] Pure rule matcher and evidence engine.
- [x] Stable reports and CLI contracts.
- [x] GitHub event/API adapter and packaged Action.
- [x] Fault injection and security regression suite.
- [x] Example policy and maintainer onboarding docs.
- [x] Local clean-room install, pack, and release-candidate verification.
- [x] Public name, package coordinates, and repository metadata selected.

## Decision log

- 2026-07-22: Use TypeScript on Node 24 for one-language CLI and Action delivery.
- 2026-07-22: Keep the engine platform-independent and free of I/O.
- 2026-07-22: Keep v1 deterministic and serverless; no model or database dependency.
- 2026-07-22: Read effective policy from the base commit and fail closed.
- 2026-07-22: Bootstrap Codex with a short map and workspace permissions; add hooks
  and skills only after a repeated workflow or failure justifies them.
- 2026-07-22: Do not claim AGM compatibility without a published interoperability
  contract and author agreement.
- 2026-07-22: Direct competitor ReviewGate validates the problem but owns general
  PR-shape heuristics; ReviewReady remains focused on repository-owned evidence
  obligations and will make no uniqueness claim.
- 2026-07-22: Support pull_request_review events and require workflow ordering
  after any checks named by policy.
- 2026-07-22: Publish under the aligned ReviewReady brand,
  `@ahoooooo/reviewready` npm package, and `ahoooooooo/reviewready` repository
  coordinates. npm rejected the unscoped name as too similar to `review-ready`;
  the scope preserves the product and CLI names without evading that safeguard.

## Release evidence

Local release-candidate evidence is recorded in
`docs/release-evidence-v1.md`. Public release remains owner-authorized work.

The evidence covers the full local gate, coverage, production Action bundle,
`npm pack --dry-run`, a tarball clean install, CLI smoke tests, GitHub event
fixtures, dependency audit, and documented limitations.
